Docker的简单使用

在工作当中，有时候会不可避免的使用ubuntu去搭建apache的服务器，但是自己的机子又是mac，所以装了docker。
docker能从网上获取各类操作系统的镜像，然后我们就可以通过这些镜像去创建一些拥有特定环境的系统，有点类似virtualenv，但是virtualenv只是虚拟环境，docker是虚拟整个系统。
由于平常用的也不是很多，所以我接下来就只是介绍一下我平常用的一些简单的操作：

  1. sudo docker pull ubuntu:16.04

    这条命令可以直接从网上获取ubuntu16.04的镜像

  2. sudo docker images

    这条命令可以查看当前你已经pull的所有镜像

  3. 当pull完整个镜像之后，你需要通过这个镜像来建立一个容器(container)。同一个镜像可以建立多个容器。

    sudo docker run -it --name="test container" --volume /app:/app ubuntu:16.04 /bin/bash

    –name 是给容器命名

    –volume 是可以把自己的目录共享到容器的系统里面，就是/app文件夹下面所有的文件，在容器里面都是可以看见和修改的

  4. 跑完run命令之后，就是已经进入了容器里面，就可以进行各种操作了，想要退出的话，可以按ctrl + D

  5. 退出来之后，可以跑：

    sudo docker ps

    查看当前正在运行的容器，这时候会发现刚刚创建的容器不在这个列表里面

    sudo docker ps -a

    查看所有的容器，刚刚创建的容器在这个列表里面

  6. 当我们需要使用或停用的时候，可以跑：

    sudo docker start/stop container_id/container_name

    – container_id就是container前面一串由数字和字母组成的字符串

  7. start一个容器之后，我们想再进入这个容器，有一个简单的方法：

    vim ~/.profile

    添加下列代码至.profile文件末尾：

    source ~/.profile

    这时候，你只需要输入dockerbash container_id/container_name就可以进入到容器之中了
